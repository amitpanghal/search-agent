| Boundary | Description |
|----------|-------------|
| shell/env ‚Üí CLI process | `ANTHROPIC_API_KEY` and the raw NL query enter the process; the key is a secret, the query is opaque text. |
| CLI/core ‚Üí Anthropic API | The only outbound network call; the query is sent as the `user` message content (no shell/fs interpolation). |
| Anthropic API ‚Üí core | Untrusted model output (`tool_use.input.plan`) crosses back in and MUST be validated before use. |
| process ‚Üí local filesystem | Reads `resolver/extractor-prompt.md` (trusted, in-repo). No cache write in this plan. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information disclosure | `ANTHROPIC_API_KEY` handling | mitigate | Key read from `process.env` only (SDK auto-read); never logged, never written to any file; `.gitignore` excludes `node_modules/` and `.cache/`; error payloads print `kind` + model input only, never the key (Task 1, Task 3). |
| T-01-02 | Tampering | Untrusted model output treated as trusted | mitigate | `QueryPlan.safeParse(tool_use.input.plan)` is the gate (D-02/D-04); a failing parse ‚Üí typed `schema_validation` failure, never coerced/executed; the plan is never `eval`/exec'd (Task 2). |
